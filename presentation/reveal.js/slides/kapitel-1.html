Kapitel 1: Einleitung (4P)

Übersicht über die Applikation (1P)

  • ULRICA = UniversaL Range and destInation CAlculator
  • Erstellt und verwaltet Fahrzeugprofile für Elektrofahrzeuge
  • Berechnet verbleibende Reichweite von Elektrofahrzeugen mit verschiedenen Effizienz-Modi (ECO, Normal, Sport)
  • Berücksichtigt verschiedene Umgebungsbedingungen (Gelände, Wetter, Temperatur)
  • Berechnet Ladezeiten an DC- und AC-Ladestationen mit verschiedenen Konditionen
  • Erstellt detaillierte Analysen zu Umwelteinflüssen auf Reichweite und Ladezeiten
  • Berücksichtigt Batterieeigenschaften (LFP, NMC, NCA) und Ladekurven

Funktionen der Applikation

Fahrzeugprofil-Management
  • Erstellung, Anzeige, Bearbeitung und Löschung von Fahrzeugprofilen
  • Konfiguration von Batteriedaten (Typ, Kapazität, Alterung)
  • Definition von Verbrauchsprofilen und Ladekurven
  • JSON-basierte Persistierung
Reichweitenberechnung
  • Berücksichtigung von Geländebedingungen
  • Anpassung an Wetterbedingungen
  • Temperatureinflüsse
  • Fahrumgebung (Stadt, Land, Autobahn)
  • Effizienz-Modi (ECO, NORMAL, SPORT)

Ladeberechnung

DC (Schnell) Laden
  • SoC-basierte Berechnungen
  • Temperatureinflüsse beim Laden
  • Leistungsreduktion basierend auf Batteriezustand
  • Detaillierte Ladezeitschätzungen
AC (Langsam) Laden
  • Verschiedene Anschlusstypen (Haushalt, Camping, Wallbox)
  • Berechnung von Effizienzverlusten
  • Temperatur-Effizenzfaktoren
  • Ladezeitprognosen

Starten der Applikation (1P)

Voraussetzungen:
  • Java 17 oder höher
  • Maven
  • Git
Schritte zum Starten:
  1. Repository klonen: git clone https://github.com/mausio/ULRICA
  2. In Projektverzeichnis wechseln: cd /path/to/ULRICA
  3. Projekt kompilieren: mvn clean compile
  4. ULRICA starten: mvn exec:java -Dexec.mainClass="org.ulrica.App"
  5. Alternativ: Ausführbare JAR erstellen: mvn clean package und dann java -jar target/ULRICA-1.0-SNAPSHOT.jar

Technischer Überblick (2P)

ULRICA wurde mit folgenden Technologien implementiert:

  • Java (JDK 17): Objektorientierte Programmiersprache - ermöglicht klare Strukturierung mit Design Patterns und Clean Architecture
  • Maven: Build-Management-Tool - vereinfacht Abhängigkeiten, Build und Tests
  • JSON/GSON: Datenaustauschformat und Java-Bibliothek für Persistenz von Fahrzeugprofilen
  • JUnit 4: Test-Framework mit 106 automatisierten Tests für Qualitätssicherung
  • JaCoCo: Code-Coverage-Tool mit einer Gesamtabdeckung von 47% (Linienabdeckung)
  • Clean Architecture: Strukturierungsprinzip für bessere Wartbarkeit und Testbarkeit
  • Design Patterns: Strategy, Adapter, Repository, Builder für besseres Design

Java (JDK 17)

Vorteile für ULRICA
  • Plattformunabhängigkeit durch JVM
  • Starke objektorientierte Programmierung
  • Typ-Sicherheit reduziert Laufzeitfehler
  • Modernes JDK 17 mit LTS-Support
  • Umfangreiche Bibliotheken verfügbar
  • Ideale Grundlage für Clean Architecture
Alternativen-Vergleich
  • Kotlin: Eleganter, aber weniger etabliert
  • C#: Ähnlich stark, aber plattformabhängiger
  • Python: Einfacher, aber weniger performant
  • JavaScript: Weniger typsicher und strukturiert
  • C++: Höhere Performance, aber komplexere Entwicklung
